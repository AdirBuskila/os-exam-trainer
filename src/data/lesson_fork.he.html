fork() — ספירת תהליכים והדפסות

fork() יוצר תהליך בן שממשיך לרוץ אחרי שורת ה-fork, עם זיכרון ומתארי קבצים (file descriptors) מועתקים. הוא מחזיר את PID של הבן לתהליך האב, 0 לתהליך הבן, ו--1 בכישלון (לא נוצר בן). תהליך הבן ממשיך לרוץ גם דרך כל fork מאוחר יותר — כך ש-k קריאות fork שאליהן מגיעים כולם נותנות 2k תהליכים.

  1. שרטטו את עץ התהליכים; סמנו מה כל fork החזיר בכל צומת (אב = PID>0, בן = 0).
  2. אם קריאות ה-fork משורשרות עם *, -, +: כל תהליך מריץ את כל הקריאות (ללא קצר חשמלי). עם ||/&&: דלגו על ה-fork השני כאשר הראשון גורם לקצר.
  3. סך התהליכים = 2(number of forks reached by all).
  4. עבור "כמה הדפסות": העריכו את ביטוי ה-if לכל תהליך; ספרו את אלה שבהם הוא אמת (לדוגמה product == 0).
  5. ספרו גם את תהליך האב עצמו — גם הוא מריץ את השורה.
if (fork() * fork() * fork() == 0) printf("fork\n");
  3 forks  -> 2^3 = 8 processes total.
  Only the original root got 3 positive PIDs (product != 0) -> no print.
  Every other process had >=1 fork return 0 (product == 0).
  prints = 8 - 1 = 7   (VERIFIED by simulation; a "4" sometimes seen is wrong for this expression)
Simpler: fork() - fork() == 0  -> "fork" prints 1 time.
  • ערכי החזרה: האב מקבל את PID של הבן (חיובי), הבן מקבל 0, כישלון נותן -1 לאב בלבד — לעולם אל תערבבו ביניהם.
  • הבן ממשיך אחרי שורת ה-fork ומריץ גם כל fork מאוחר יותר — הוא אינו מתחיל מחדש מ-main.
  • printf ללא \n נשאר בחוצץ של libc; ה-fork מעתיק אותו, ולכן הוא מודפס פעמיים ביציאה. תיקון: fflush או \n.
  • קצר חשמלי של || / && יכול לדלג על fork — פחות תהליכים מאשר 2k הגולמי. */-/+ רגילים אינם גורמים לקצר חשמלי (אך סדר ההערכה ב-C אינו מוגדר).
  • הזיכרון מועתק, לא משותף (COW): אחרי fork, שינוי במשתנה של האב אינו נראה לבן.
  • תמיד כללו את האב בספירה; טעות נפוצה היא לספור רק את הבנים.
