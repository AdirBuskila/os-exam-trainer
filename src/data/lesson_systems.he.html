מושגי מערכות: אוצר המילים של מערכת ההפעלה

זהו הנושא הקטן והקל ביותר — שינון טהור של קומץ הגדרות. זהו אוצר המילים של התמונה הגדולה במערכות הפעלה: כיצד מספר מעבדים חולקים משאבים, כיצד נקראת מערכת הפעלה עם הרבה תהליכים, כיצד מודדים ביצועים, מה עושים מנהלי התקנים, וקטגוריות הזמן-אמת. אין כאן חישובים ואין דיאגרמות. כל כרטיס פשוט ממפה הגדרה אחת למונח אחד, ולכן כל תפקידכם הוא להתאים את ההגדרה למונח שלה.

מונחמשמעות פשוטהמונח עברי
Tightly coupledה־CPUs חולקים זיכרון ראשי אחד (RAM)צימוד הדוק
Loosely coupledלכל CPU יש זיכרון משלו; הם מתקשרים דרך רשתצימוד רופף
Multiprogramming / time-sharingמספר תהליכים מוחזקים בזיכרון בו-זמנית; ה־CPU מריץ אותם לסירוגיןריבוי תכנות / שיתוף זמן
Throughputכמה תהליכים מסתיימים ליחידת זמן (מדד ביצועים)תפוקה
Benchmarkעומס עבודה תקני המורץ כדי למדוד את ביצועי המחשבמבחן ביצועים
Device driverקוד kernel המתווך בין החומרה ליישומיםמנהל התקן
Hard real-timeתמיד עומד ב־deadline (מערכות בקרה קריטיות)זמן אמת קשיח
Soft real-timeעומד ב־deadline רוב הזמן (לא תמיד); החמצה נסבלתזמן אמת רך

מכיוון שכל כרטיס הוא התאמה של הגדרה למונח, רשימת החלטות קצרה היא כל מה שדרוש כדי לענות במהירות ובדיוק.

  1. צימוד: הרכיב שמשותף בין ה־CPUs בצימוד הדוק אך נפרד לכל CPU בצימוד רופף הוא הזיכרון הראשי (RAM).
  2. זמן אמת: המילים רוב הזמן / לא תמיד מצביעות על soft real-time; המילה תמיד מצביעה על hard real-time.
  3. ביצועים: throughput הוא משימות שהסתיימו ליחידת זמן — מדד מערכתי, לא שעון לכל תהליך.
  4. גישה לחומרה: device driver מתווך בין החומרה ליישומים, ומעניק למערכת ההפעלה דרך אחידה לתקשר עם כל התקן.
Q: "Component shared tightly but separate loosely?"   -> RAM
Q: "Meets deadline most of the time?"                 -> Soft real-time
Q: "Always meets the deadline?"                       -> Hard real-time
Q: "How many jobs finish per unit time?"              -> Throughput
Q: "Mediates between hardware and applications?"       -> Device driver

מה נשאל במבחן

נקבו במונח לפי ההגדרה שלו — זה כל הנושא. להלן כל ששת הכרטיסים החוזרים עם התשובות המאומתות שלהם.

כרטיסשאלה (מקוצרת)תשובה
2017A Q1הרכיב המשותף (הדוק) אך נפרד לכל CPU (רופף)זיכרון ראשי (RAM)
2019B Q3מדד: כמה תהליכי benchmark מסתיימים בזמן קבועThroughput
2020B Q1מערכת המחזיקה תהליכים רבים בזיכרון, ומריצה אותם לסירוגיןMultiprogramming / time-sharing

2024A Q1מערכת העומדת ב־deadlines רוב הזמן (לא תמיד)Soft real-time
המלכודות שעולות נקודות:
  • צימוד הדוק = RAM משותף; צימוד רופף = זיכרון פרטי לכל CPU. הצימוד עוסק בשיתוף זיכרון, לא במספר ה־CPU.
  • Multiprogramming אינו דורש מספר מעבדים: משמעו תהליכים רבים השוכנים בזיכרון ומורצים לסירוגין — CPU אחד מספיק.
  • Throughput מול turnaround: throughput = משימות שהסתיימו ליחידת זמן (קצב מערכתי); turnaround/זמן המתנה = שעונים לכל תהליך מנושא התזמון.
  • Device driver מול system call: ה־driver הוא קוד kernel להתקן אחד; system call היא נקודת הכניסה מ־userspace אל ה־kernel. שכבה שונה.
  • Soft = רוב הזמן, hard = תמיד: "רוב הזמן (לא תמיד)" פירושו soft real-time; "תמיד / מובטח" פירושו hard real-time.
