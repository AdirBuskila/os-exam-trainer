תהליכון (Thread / lightweight process) — היחידה הבסיסית לניצול ה-CPU. לתהליכון יש מחסנית (stack) ואוגרים (registers) משלו (וכן PCB בקרנל); הוא חולק עם התהליכונים האחרים שלו את מקטע הקוד (text), הנתונים (data) והקבצים הפתוחים. החבילה המשותפת הזו נקראת task. תהליך מסורתי (heavyweight) שקול ל-task עם תהליכון אחד בלבד.

למה תהליכונים?

  • בזמן שתהליכון (שרת) אחד חסום/ממתין, תהליכון אחר באותה משימה יכול לרוץ.
  • יישומים שחולקים buffer משותף (יצרן–צרכן) מרוויחים משימוש בתהליכונים.
  • הם מאפשרים לתהליכים סדרתיים לבצע קריאות מערכת חוסמות תוך השגת מקביליות (parallelism).

סוגי תהליכונים

  • תהליכונים נתמכי-קרנל (Kernel-supported).
  • תהליכונים ברמת המשתמש (User-level) — נתמכים מעל הקרנל באמצעות אוסף קריאות ספרייה.
  • תהליכוני Anderson — כמו נתמכי-קרנל, אך שולחים אירועי תזמון אל ה-task במקום לפרש אותם בעצמם.

מודלים של ריבוי-תהליכונים

  • Many-to-One — תהליכוני משתמש רבים → תהליכון קרנל אחד; במערכות שאינן תומכות בתהליכוני קרנל (לדוגמה Project Andrew של CMU).
  • One-to-One — כל תהליכון משתמש → תהליכון קרנל משלו (Windows 95/98, OS/2).
  • Many-to-Many — תהליכוני משתמש רבים ↔ תהליכוני קרנל רבים; גישה היברידית (Solaris 2, Linux, Windows NT/2000).

Solaris 2

  • LWP (lightweight process) = רמת הביניים שבין תהליכוני המשתמש לתהליכוני הקרנל.
  • תהליכון משתמש: צריך רק מחסנית + program counter, ללא מעורבות קרנל → החלפה מהירה.
  • תהליכון קרנל: מבנה נתונים קטן + מחסנית; החלפה מהירה (ללא שינוי מיפוי הזיכרון). החלפת LWP איטית יותר (PCB עם נתוני אוגרים/חשבונאות/זיכרון).
