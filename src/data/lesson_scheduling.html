CPU Scheduling

There is one CPU but many processes sitting ready to run, so something must decide who goes next. A scheduling algorithm is just a rule for picking the next process (and, for Round-Robin, for how long it runs). Exam questions come in exactly two flavours: name a concept (a definition, a preemptive example, an OS quirk) or compute the average waiting time from a given set of arrivals and bursts. Master one method and every compute question becomes the same drill.

Glossary

TermHebrewMeaning
CPU burstפרץ חישובA stretch of pure computing between two I/O requests. "Adds two register values" is a CPU burst.
Waiting timeזמן המתנהTotal time spent in the ready queue, not running and not doing I/O. Equals Turnaround minus Burst.
Turnaround timeזמן סבבFrom arrival to completion: the process's whole lifetime in the system. Equals Completion minus Arrival.

The universal method

  1. Build the table: Process, Arrival, Burst (add a Remaining column for SRTF and RR).
  2. Draw the Gantt chart by the algorithm's rule, re-checking at every arrival or completion (and every quantum for RR).
  3. Read off the Completion time of each process from the chart.
  4. Compute Turnaround = Completion − Arrival.
  5. Compute Waiting = Turnaround − Burst (the universal formula; it works even when a job runs in pieces).
  6. Average waiting time = sum of waiting times divided by the number of processes.

Worked examples (verified answers)

FCFS  2021B Q5   (P1 arr0 b7, P2 arr1 b7, P3 arr2 b7)
P1 |#######|              0-7
P2         |#######|      7-14
P3                 |#######|   14-21
Completion: P1=7, P2=14, P3=21
Waiting = Turnaround - Burst
P1: (7-0)-7  = 0
P2: (14-1)-7 = 6
P3: (21-2)-7 = 12
Average = (0 + 6 + 12) / 3 = 6 ms

P1: (20-0)-9 = 11
P2: (12-1)-7 = 4
P3: (6-2)-4  = 0
Average = (11 + 4 + 0) / 3 = 5 ms

  • Three different clocks. Waiting is time stuck in the ready queue; Response is time until the first output; Turnaround is arrival to completion. Waiting never includes running time.
  • Use Waiting = Turnaround − Burst for RR and SRTF. Those jobs run in pieces, so "Start − Arrival" fails; only the universal formula is safe.
  • RR ordering rule. When a slice ends, enqueue jobs that arrived during that slice first, then the just-preempted job behind them. Putting the preempted job back first breaks the answer.
  • Starvation is fixed by aging. SJF, SRTF, and Priority can starve long or low-priority jobs; slowly raising a waiting process's priority (aging) prevents it. RR never starves.
  • Windows focus window gets a 3x (triple) quantum. The process owning the focus window runs three times longer per slice.
  • Linux = epoch + Multilevel Queue. One round over all CPU-requesting processes is an epoch; the course's expected answer for Linux's scheduling approach is Multilevel Queue. (Bonus identity: RR with an infinite quantum equals FCFS.)
