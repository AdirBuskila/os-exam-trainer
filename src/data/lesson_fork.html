Fork Counting — Processes & Prints

The whole topic is one problem: fork() duplicates the running process, and both copies continue from the same next line. Given a short C program with one or more fork calls, the exam asks you to count how many processes exist and/or how many times a line is printed. Draw the tree, label the returns, apply the condition, count — every Q9 falls to this.

  1. Draw the process tree. Start with 1 process; each fork reached by a branch doubles it. The leaves are the total processes.
  2. Label each process with the return value it saw at every fork (parent got PID>0, child got 0).
  3. Apply the condition. With &&/|| a fork can be skipped (smaller tree); with */-/+/== all forks run. Test the guard per process to mark who prints.
  4. Count the marked processes (and remember to include the original parent).
Plain forks: N calls reached by everyone -> 2^N processes.
  fork(); fork(); fork(); printf("x");   // 2^3 = 8 prints

fork() * fork() * fork() == 0   ->  prints 7   (2024A Q9)
  No short-circuit: all 3 forks run -> 8 processes.
  Product != 0 only for the all-parent process (3 positive PIDs).
  The other 8 - 1 = 7 had >=1 fork return 0 -> product == 0 -> print.

fork() - fork() == 0   ->  prints 1   (2024B Q9)
  No short-circuit: 2 forks run -> 4 processes.
  first - second == 0 only when both returns are equal: 0 - 0 = 0
  -> the all-child process only -> 1 print.

Rule of thumb:  *...==0 -> 2^N - 1 print ;  -...==0 -> 1 prints.

  • Return-value direction: parent gets the child PID (positive), child gets 0, failure gives -1 to the parent only — never mix these up.
  • && / || short-circuit can skip a fork, so the tree is smaller than the raw 2N. Arithmetic and comparison (* - + ==) never short-circuit — all their forks run.
  • Code after while(1) or exit() never runs — forks placed after it create no processes.
  • The two one-liners: for *...==0 the all-parent process is the only nonzero, so 2N−1 print; for -...==0 the all-child process is the only match, so just 1 prints.
  • Always count the parent and remember the child continues after the fork line (it does not restart at main) — a common slip is counting only children.
