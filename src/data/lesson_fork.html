fork() — Counting Processes & Prints

fork() creates a child that resumes after the fork line, with copied memory and file descriptors. It returns the child PID to the parent, 0 to the child, and -1 on failure (no child created). The child keeps running every later fork too — so k fork calls reached by all give 2k processes.

  1. Draw the process tree; mark what each fork returned at each node (parent = PID>0, child = 0).
  2. If forks are chained with *, -, +: every process runs all calls (no short-circuit). With ||/&&: skip the second fork when the first short-circuits.
  3. Total processes = 2(number of forks reached by all).
  4. For "how many prints": evaluate the if expression per process; count those where it is true (e.g. product == 0).
  5. Count the parent itself too — it executes the line as well.
if (fork() * fork() * fork() == 0) printf("fork\n");
  3 forks  -> 2^3 = 8 processes total.
  Only the original root got 3 positive PIDs (product != 0) -> no print.
  Every other process had >=1 fork return 0 (product == 0).
  prints = 8 - 1 = 7   (VERIFIED by simulation; a "4" sometimes seen is wrong for this expression)
Simpler: fork() - fork() == 0  -> "fork" prints 1 time.
  • Return values: parent gets child PID (positive), child gets 0, failure gives -1 to parent only — never mix these up.
  • Child continues after the fork line and executes every later fork too — it does not restart at main.
  • printf without \n stays in the libc buffer; fork copies it, so it prints twice at exit. Fix: fflush or \n.
  • || / && short-circuit can skip a fork — fewer processes than the raw 2k. Plain */-/+ do not short-circuit (but C evaluation order is undefined).
  • Memory is copied, not shared (COW): after fork, a parent's variable change is invisible to the child.
  • Always include the parent in the count; a common slip is counting only children.
