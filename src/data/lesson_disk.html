Storage hierarchy (fast/expensive → slow/cheap): Registers → Cache → RAM → SSD → HDD → Tape. Cheapest = HDD (or tape); fastest I/O destination = RAM/SSD.

  1. Identify the keyword in the question (spool / DMA / cheapest / "without moving head" / swap / random access).
  2. Spool sits on → hard disk (HDD). DMA writes to → RAM. Cheapest → HDD.
  3. "File on the same ___" / "without moving the read head" → Cylinder.
  4. Linux stores evicted/rolled-out processes in → Swap partition.
  5. FAT-32 slow vs UFS → FAT walks a linear pointer chain; UFS uses an i-node (direct + indirect).
Q: "To modify the last byte of a 100MB file, why is FAT-32 much slower than UFS?"
FAT-32: reach block n => walk n-1 pointers in a chain (linear)  -> slow
UFS:    i-node direct/indirect/double-indirect => constant # of accesses -> fast
A (<=5 words): FAT walks linear pointer chain
  • Spool = stable queue on the HARD DISK, not in cache/RAM. Cache is a temporary fast copy; spool persists on disk.
  • DMA bypasses the CPU — it writes to RAM, NOT to the CPU. That is the whole point of DMA.
  • Cylinder vs Track: a file should sit on the same Cylinder (no head movement), not the same track. Cylinder = tracks at one radius across all platters.
  • Swap = whole process (roll out/in); Paging = single 4KB page. Do not mix them.
  • Cheapest = HDD in this course's answer (tape is cheaper in theory, but exam answer is HDD).
  • FAT-32 is slow because of the linear chain, not because blocks are far apart. UFS speed comes from the i-node's indirect pointers.
