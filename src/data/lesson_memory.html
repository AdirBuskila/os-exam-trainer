Memory: Virtual Memory, Paging & the Page-Fault Path

Virtual memory gives each process an isolated address space larger than RAM. The MMU translates virtual→physical via a per-process page table; the TLB caches recent translations. Most exam points here are one-word concept recalls plus tiny power-of-2 frame/access math.

Frame / page & TLB-miss math

  1. Express RAM and page size as powers of 2. #frames = RAM / page_size → subtract exponents.
  2. offset bits = log2(page_size); page-number bits = address_bits − offset bits.
  3. #pages (or #frames) = 2^(page-number bits).
  4. Multilevel virtual space = (entries per level)^levels × page_size.
  5. TLB hit = 1 memory access (the page). TLB miss = levels + 1 accesses (walk all N tables, then the page).
42-bit virtual address, page size 1MB = 2^20
  page-number bits = 42 - 20 = 22  =>  #frames = 2^22
3-level multilevel paging, TLB miss:
  accesses = 3 (tables) + 1 (page) = 4
Close-but-precise pitfalls
  • Who sets which bit: CPU/MMU sets the Reference and Dirty bits (automatic on access/write). The OS sets the Present/Valid bit and clears the reference bit (e.g. in Clock).
  • TLB miss = N+1 accesses, not N — never forget the access to the page itself.
  • Frames ≠ pages: physical frame count comes from RAM size; virtual page count comes from the address space. Don't mix them.
  • Page vs Swap: paging moves a single page (4KB); swap (roll out/in) moves an entire process. Linux evicted pages go to the swap partition.
  • Internal (not external) fragmentation is the paging one — waste inside the last page; pure paging has no external fragmentation.
  • COW is what makes fork() cheap: pages are shared read-only and copied only when one side writes — not copied up front.
  • Multilevel table saves memory (don't hold the whole table) but costs extra accesses per translation.
  • Belady's anomaly (more frames → more faults) belongs to FIFO only; LRU and OPT are stack algorithms and never suffer it.
  • Answer in English term, ≤5 words (e.g. "Page fault", "Copy-on-Write") — Hebrew spelling is risky.
