תזמון CPU = המתזמן לטווח הקצר בוחר איזה תהליך במצב READY יקבל את ה-CPU. יש לדעת לכל אלגוריתם את שמו, האם הוא מפנה (preemptive), ואת הכשל הקטלני היחיד שלו. בשאלות השלמה במבחן יש לכתוב את המונח באנגלית; נוסחאות המדדים ותרשים Gantt אחד מכסים את השאר.

מתזמן לטווח קצרCPU schedulerמתזמן לטווח קצרבוחר runnable→running; רץ כל כמה ms.
  1. רשום כל תהליך עם Arrival ו-Burst (ו-Priority אם ניתן).
  2. החל את כלל האלגוריתם בכל אירוע (הגעה חדשה או השלמה). SJF/Priority = בחר את הטוב ביותר מבין מי שכבר הגיעו; SRTF/RR = בדוק מחדש בכל אירוע/קוונטום.
  3. צייר את תרשים ה-Gantt (ציר זמן עם מי שרץ).
  4. לכל תהליך: Turnaround = Completion − Arrival; Waiting = Turnaround − Burst; Response = First-CPU − Arrival.
  5. ממוצע = סכום / n. שוויון ב-burst → שובר לפי FCFS (הגעה). זכור: מספר קטן יותר = עדיפות גבוהה יותר.
P1 burst 24, P2 burst 3, P3 burst 3 (all arrive at 0)

FCFS  | P1            | P2 | P3 |   wait: 0, 24, 27  -> avg 17
      0              24   27   30

SJF   | P2 | P3 | P1            |   wait: 6, 0, 3    -> avg 3  (optimal)
      0    3    6              30

RR Q=4| P1 |P2 |P3 | P1 P1 P1 P1 P1 |  wait avg ~ 5.67
  • SJF הוא לא-מפנה; SRTF הוא מפנה. שני אלגוריתמים שונים — אין לערבב.
  • RR עם קוונטום = אינסוף (או Q ≥ ה-burst הארוך ביותר) מתנהג כמו FCFS. Q קטן מדי → CPU מבוזבז על החלפות הקשר; Q גדול מדי → כמו FCFS.
  • ה-convoy effect שייך ל-FCFS (עבודה ארוכה אחת חוסמת את כולם). SJF/SRTF נותנים את זמן ההמתנה הממוצע המינימלי, אך רק על קבוצה נתונה.
  • הרעבה היא תוצאה, לא אלגוריתם. היא פוגעת ב-SJF/SRTF (עבודות ארוכות) וב-Priority (עדיפות נמוכה); הפתרון הוא aging.
  • זמן ההמתנה משתמש ב-burst, לא בזמן הכולל. Waiting = Turnaround − Burst. Response = הפעם הראשונה על ה-CPU, לא ההשלמה.
  • Priority: מספר קטן יותר = עדיפות גבוהה יותר בקורס זה. טווח nice מ-−20 (גבוה) … +19 (נמוך).
  • Linux = Multilevel Queue (FIFO real-time, RR real-time, לא-זמן-אמת/מבוסס-epoch). מחזור מלא אחד = Epoch; הקצאה שלא נוצלה נושאת חצי ל-epoch הבא.
  • Windows: חוט של חלון החזית מקבל קוונטום משולש; לאחר I/O, חיזוק העדיפות הוא ביחס הפוך למהירות ההתקן (התקן איטי → חיזוק גדול יותר), ואז דועך בכל קוונטום.
  • RR נשלף על ידי פסיקת הטיימר (סוף הקוונטום), לעולם לא מרצון.
