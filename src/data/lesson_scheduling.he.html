תזמון CPU

יש CPU אחד אך תהליכים רבים שממתינים במצב מוכן לרוץ, ולכן משהו חייב להחליט מי רץ הבא. אלגוריתם תזמון הוא פשוט כלל לבחירת התהליך הבא (ובמקרה של Round-Robin גם למשך כמה זמן ירוץ). שאלות המבחן מגיעות בשני סוגים בלבד: לנקוב בשם של מושג (הגדרה, דוגמה לאלגוריתם מפנה, מאפיין של מערכת הפעלה) או לחשב את זמן ההמתנה הממוצע מתוך קבוצת הגעות ו-bursts נתונה. מי ששולט בשיטה אחת הופך כל שאלת חישוב לאותו תרגיל בדיוק.

מילון מונחים

מונחעבריתמשמעות
CPU burstפרץ חישובקטע של חישוב טהור בין שתי בקשות I/O. "מחבר שני ערכי אוגרים" הוא CPU burst.
Waiting timeזמן המתנהסך הזמן שהתהליך מבלה בתור המוכנים, כשאינו רץ ואינו מבצע I/O. שווה ל-Turnaround פחות Burst.
Turnaround timeזמן סבבמההגעה ועד ההשלמה: כל אורך חיי התהליך במערכת. שווה ל-Completion פחות Arrival.
Response timeזמן תגובהמההגעה ועד שהתהליך מפיק את הפלט הראשון שלו (הפעם הראשונה שהוא רץ).
Throughputתפוקהמספר התהליכים שהושלמו ליחידת זמן.
Preemptiveנשלף / מפנההמתזמן יכול לשלוף תהליך רץ בחזרה למצב מוכן. דוגמאות: SRTF ו-Round-Robin.
Non-preemptiveלא נשלףתהליך שנבחר מחזיק ב-CPU עד שמסיים את ה-burst שלו או נחסם. דוגמאות: FCFS ו-SJF.
Quantumקוונטוםפרוסת הזמן הקבועה שכל תהליך מקבל ב-Round-Robin לפני שהוא נשלף.
Starvationהרעבהתהליך שלעולם אינו זוכה לרוץ (קורבן של Priority או SJF/SRTF). הפתרון הוא aging.

השיטה האוניברסלית

  1. בנה את הטבלה: תהליך, Arrival, Burst (הוסף עמודת Remaining עבור SRTF ו-RR).
  2. צייר את תרשים ה-Gantt לפי כלל האלגוריתם, תוך בדיקה מחדש בכל הגעה או השלמה (וכל קוונטום ב-RR).
  3. קרא את זמן ה-Completion של כל תהליך מהתרשים.
  4. חשב Turnaround = Completion − Arrival.
  5. חשב Waiting = Turnaround − Burst (הנוסחה האוניברסלית; היא עובדת גם כשעבודה רצה בחלקים).
  6. זמן ההמתנה הממוצע = סכום זמני ההמתנה חלקי מספר התהליכים.

דוגמאות פתורות (תשובות מאומתות)

FCFS  2021B Q5   (P1 arr0 b7, P2 arr1 b7, P3 arr2 b7)
P1 |#######|              0-7
P2         |#######|      7-14
P3                 |#######|   14-21
Completion: P1=7, P2=14, P3=21
Waiting = Turnaround - Burst
P1: (7-0)-7  = 0
P2: (14-1)-7 = 6
P3: (21-2)-7 = 12
Average = (0 + 6 + 12) / 3 = 6 ms

SRTF  2022A Q4   (P1 arr0 b9, P2 arr1 b7, P3 arr2 b4)
P1 |#|                        0-1
P2   |#|                      1-2   (P2 rem7 < P1 rem8, preempt)
P3     |####|                 2-6   (P3 smallest remaining)
P2         |######|           6-12
P1               |########|   12-20
Completion: P1=20, P2=12, P3=6
Waiting = (Completion - Arrival) - Burst
P1: (20-0)-9 = 11
P2: (12-1)-7 = 4
P3: (6-2)-4  = 0
Average = (11 + 4 + 0) / 3 = 5 ms

RR q=4  2021A Q5   (three 7 ms jobs, all arrive t=0)
P1|####|P2|####|P3|####|P1|###|P2|###|P3|###|
  0    4      8      12   15    18    21
Completion: P1=15, P2=18, P3=21
Waiting = Turnaround - Burst
P1: 15-7 = 8
P2: 18-7 = 11
P3: 21-7 = 14
Average = (8 + 11 + 14) / 3 = 11 ms

RR q=4  2019A Q4   (bursts 9,3,7,1; all arrive t=0)
P1|####|P2|###|P3|####|P4|#|P1|####|P3|###|P1|#|
  0    4     7      11  12   16     19   20
Completion: P1=20, P2=7, P3=19, P4=12
Waiting = Turnaround - Burst
P1: 20-9 = 11
P2: 7-3  = 4
P3: 19-7 = 12
P4: 12-1 = 11
Average = (11 + 4 + 12 + 11) / 4 = 9.5 ms

RR q=4  2022B Q4   (P1 arr0 b9, P2 arr1 b7, P3 arr3 b4)  -- staggered, the hard one
0-4    P1   rem5   arrivals during slice: P2, P3  -> queue [P2, P3, P1]
4-8    P2   rem3                                  -> queue [P3, P1, P2]
8-12   P3   rem0 (done)                           -> queue [P1, P2]
12-16  P1   rem1                                  -> queue [P2, P1]
16-19  P2   rem0 (done)                           -> queue [P1]
19-20  P1   rem0 (done)                           -> queue []
Completion: P1=20, P2=19, P3=12
Waiting = (Completion - Arrival) - Burst
P1: (20-0)-9 = 11
P2: (19-1)-7 = 11
P3: (12-3)-4 = 5
Average = (11 + 11 + 5) / 3 = 9 ms

ORDERING RULE: arrivals during a slice queue BEFORE the just-preempted job.
RR with an infinite quantum = FCFS (nobody is ever preempted).

  • שלושה שעונים שונים. המתנה היא הזמן התקוע בתור המוכנים; תגובה היא הזמן עד הפלט הראשון; סבב הוא מההגעה ועד ההשלמה. ההמתנה לעולם אינה כוללת זמן ריצה.
  • השתמש ב-Waiting = Turnaround − Burst עבור RR ו-SRTF. עבודות אלה רצות בחלקים, ולכן "Start − Arrival" נכשל; רק הנוסחה האוניברסלית בטוחה.
  • כלל הסידור של RR. כשפרוסה מסתיימת, הכנס לתור תחילה את העבודות שהגיעו במהלך אותה פרוסה, ורק אחריהן את העבודה שזה עתה נשלפה. החזרת העבודה הנשלפת ראשונה שוברת את התשובה.
  • הרעבה נפתרת באמצעות aging. SJF, SRTF ו-Priority עלולים להרעיב עבודות ארוכות או בעדיפות נמוכה; העלאה הדרגתית של עדיפות תהליך ממתין (aging) מונעת זאת. RR לעולם אינו מרעיב.
  • חלון החזית ב-Windows מקבל קוונטום פי 3 (משולש). התהליך שבבעלותו חלון החזית רץ פי שלושה זמן בכל פרוסה.
  • Linux = epoch + Multilevel Queue. סבב אחד על כל התהליכים המבקשים CPU הוא epoch; התשובה הצפויה בקורס לגישת התזמון של Linux היא Multilevel Queue. (זהות בונוס: RR עם קוונטום אינסופי שקול ל-FCFS.)
