Page Replacement & Thrashing

Under demand paging RAM holds only some pages. When a page fault needs a new page but every frame is full, the OS must pick a victim page to evict. The page-replacement algorithms are just the different rules for choosing that victim — and thrashing is what happens when the system gets it badly wrong. Answer in one or two words, in English.

How to answer

  1. Optimal evicts the page used farthest in the future; LRU evicts the least recently used page in the past. The "future vs past" wording is the tell.
  2. FIFO evicts the oldest-loaded page, and it is the only algorithm that suffers Belady's anomaly (LRU and OPT are stack algorithms, immune).
  3. Clock: hardware sets the reference bit to 1 on access; the OS clears it to 0 while sweeping for a victim. A referenced page gets a second chance (bit cleared, skipped). Finding a victim is O(N) in the worst case.
  4. Local allocation: a faulting process may evict only its own pages, so bringing in a page for X evicts a page of X.
  5. Thrashing comes from a shortage of frames (processes need more frames than exist). The fix is to lower the degree of multiprogramming.
Clock sweep (hand looks for reference bit R = 0):
  page A  R=1  ->  second chance: OS clears R to 0, advance
  page B  R=1  ->  second chance: OS clears R to 0, advance
  page C  R=0  ->  VICTIM (evict C)
  worst case: hand visits all N frames  ->  O(N)

FIFO + Belady: more frames can give MORE faults (FIFO only).
Optimal: evict the page whose next use is farthest in the FUTURE.

Close-but-precise traps

  • Optimal = future, LRU = past. Optimal evicts the page used farthest in the future; LRU evicts the least recently used page. "Future" vs "past" wording decides the answer.
  • Belady's anomaly is FIFO only. If more frames caused more faults, the algorithm is FIFO. LRU and OPT are stack algorithms and never suffer it.
  • Clock bits: hardware sets the reference bit on access; the OS clears it during the victim search. A referenced page gets a second chance (bit cleared, skipped), and the search is O(N).
  • Local allocation evicts the faulting process's own page; global allocation may evict any process's page.
  • Thrashing = too many frames demanded for the RAM available, so the OS constantly swaps. The fix is lowering the degree of multiprogramming, not changing the algorithm.
